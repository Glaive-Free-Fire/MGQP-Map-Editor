## Новое в версии v1.5.15

### 1. Обработка больших файлов (Система чанков)
*   **Чередование частей**: Файлы, превышающие 5000 строк, теперь автоматически разбиваются на «чанки» (части). Это позволяет избежать зависаний браузера при отрисовке огромного количества элементов в DOM.
*   **Синхронизация границ**: Японский оригинал теперь нарезается строго по тем же границам `CommonEvent`, что и русский перевод, что исключает рассинхронизацию строк в конце каждой части.
*   **Навигация**: Добавлены кнопки «Пред. Часть» и «След. Часть» с индикатором текущего положения (например, «Часть 1/3»).
*   **Корректное сохранение**: При нажатии «Сохранить» редактор автоматически склеивает все части в один целый файл.

### 2. Защита данных и Автосохранение
*   **Кэширование сессии (localStorage)**: Реализована система фонового автосохранения каждые 5 минут. Если вы случайно закроете вкладку или произойдет сбой, при следующем открытии появится окно восстановления сессии.
*   **Защита от перезаписи**: Исправлен баг, при котором открытие нового файла могло привести к перезаписи его начала данными из предыдущей активной сессии. Теперь при загрузке нового файла кэш полностью сбрасывается.

### 3. Улучшенная детекция ошибок и валидация
*   **Слипание структурных строк**: Добавлена проверка на отсутствие пустой строки перед блоками `CommonEvent` и `Page`. Редактор автоматически исправляет это «слипание» при сохранении файла для обеспечения корректной работы движка.
*   **Детекция незакрытых кавычек**: Добавлена проверка на нечётное количество кавычек `"` внутри блоков `ShowText`. Ошибочные блоки подсвечиваются красным.
*   **Display Name**: В функции генерации строк исправлено дублирование заголовка `Display Name`. Теперь он добавляется только в самую первую часть (чанк) файла.

### 4. Оптимизация интерфейса (Removal)
*   **Удален «Режим переноса»**: Убрана кнопка `wrapModeBtn` и связанная с ней логика (Остаточный/Равный).
*   **Удалена кнопка "\...\" (wrapSlashesBtn)**: Полностью вырезана логика отслеживания фокуса и функции автоматической расстановки слешей, что снизило нагрузку на процессор при редактировании.
*   **Удалены кнопки Undo/Redo**: Устаревшие кнопки «↺» и «↻» удалены из интерфейса для чистоты кода.

### 5. Исправление багов
*   **Чистка кода в test_mode.js**: Из цикла поиска ID предметов удалён ошибочно вставленный дублирующий блок кода.
*   **Сброс состояния**: При загрузке новой карты теперь корректно сбрасываются старые ошибки, имя карты и флаги изменений.
